# Implementation Plan: Landing Page

**Branch**: `004-landing-page` | **Date**: 2026-03-03 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/004-landing-page/spec.md`

## Summary

Build a bilingual (EN/ZH) landing page at `/en` and `/zh` that introduces the platform's two B2B lead-generation services (Form Submission and Inquiry Widget). The page dogfoods both services: a "Contact Us" form submits via the platform's own form endpoint, and the floating Inquiry Widget is embedded live. The root URL `/` auto-detects browser language and redirects. Comprehensive SEO (structured data, hreflang, OG/Twitter meta) and AI-crawler friendliness are built in. Authenticated users visiting the landing page are redirected to the dashboard.

Design style: clean and minimal, with strong headlines and clear service selling points. No heavy graphics — relies on typography, whitespace, and subtle accents from the existing shadcn/ui design system.

## Key Implementation Details

### 1. Routing & Middleware

**Middleware** (`frontend/middleware.ts`):
- Matcher: `['/', '/en', '/zh']`
- `/` → read `Accept-Language` header → redirect 302 to `/zh` (if starts with `zh`) or `/en` (default)
- `/en`, `/zh` → check `pb_auth` cookie existence → if present, redirect 302 to `/dashboard`
- Sets `x-locale` response header for root layout to read

**Dynamic segment** (`app/[locale]/page.tsx`):
- `generateStaticParams()` returns `[{locale:'en'}, {locale:'zh'}]`
- `dynamicParams = false` — only `/en` and `/zh` are valid; others get 404
- Server component for full SEO (HTML rendered on server)

**Root page** (`app/page.tsx`):
- Removed or emptied — middleware handles `/` redirect before this file is reached

### 2. i18n Dictionary System

```typescript
// src/lib/dictionaries.ts
const dictionaries = {
  en: () => import('../dictionaries/en.json').then(m => m.default),
  zh: () => import('../dictionaries/zh.json').then(m => m.default),
};
export const getDictionary = async (locale: 'en' | 'zh') => dictionaries[locale]();
```

### 3. SEO Implementation

**Per-locale metadata** (`app/[locale]/layout.tsx` using `generateMetadata()`):
- `title`: localized page title
- `description`: localized meta description
- `alternates.canonical`: self-referencing canonical URL
- `alternates.languages`: `{ 'en': '/en', 'zh': '/zh' }` for hreflang
- `openGraph`: title, description, URL, type, locale
- `twitter`: card, title, description

**Structured data** (JSON-LD in layout):
- `SoftwareApplication` for the product
- `Organization` for the brand
- Embedded as `<script type="application/ld+json">`

**robots.txt** (`public/robots.txt`):
```
User-agent: *
Allow: /

User-agent: GPTBot
Allow: /

User-agent: ClaudeBot
Allow: /
```

### 4. Contact Form (Client Component)

- Client component (`'use client'`) with React state for form fields, loading, success, error
- Email validation via HTML5 `type="email"` + pattern check before submission
- Submits via `fetch()` to `${NEXT_PUBLIC_POCKETBASE_URL}/api/submit/${NEXT_PUBLIC_LANDING_FORM_ID}`
- Uses `redirect: 'manual'` to handle the 302 response as success
- Shows loading spinner on button during submission
- Shows success message after submission
- Shows error message with retry option on failure

### 5. Widget Embed

- Uses Next.js `<Script>` component with `strategy="afterInteractive"`
- Sets `window.FormHandler.widgetId` and `window.FormHandler.apiBase` before loading
- Widget script URL from `NEXT_PUBLIC_WIDGET_URL`
- Widget ID from `NEXT_PUBLIC_LANDING_WIDGET_ID`
